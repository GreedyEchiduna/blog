## 练习

1. [rustlings](https://github.com/rust-lang/rustlings)

rustlings 是 Rust 官方推出的一个练习项目, 里面包含了 Rust 的大部分语法, 适合新手练习, 也适合老手复习. 值得一提的是, rustlings 本身有提供一个相当有趣的 Rust 编写的命令行工具用来跟踪检查练习进度, 以及搜寻提示, 测试答案.

推荐在学习 Rust book 的同时, 在 rustlings 练习对应的章节中. 值得注意的是, rustlings 本身并不提供讲解和参考答案, 而是在每个`exercises`中的`README.md`给出相应的参考资料, 有相当一部分都是标准库. 因此, 完成 rustlings 的同时, 参阅[Rust std](https://doc.rust-lang.org/std/)是必不可少的

不过 rustlings 并不是十全十美的, 在博主完成 quiz3.rs 的时候, 就十分困惑作者究竟想要大家使用什么方法实现, 就算阅读了提示, 这个问题也相当地迷惑. 其中有一些练习模块也需要别的模块的知识(尤其是多线程或者是智能指针部分), 这当然也有 Rust 特性环环相扣的原因在. 所以, 在完成 rustlings 之前, 对 Rust 有宏观基本的理解是最好的.

rustlings 练习的深度还是相较浅的, rustlings 的答案可以参照[博主的实现](https://github.com/GreedyEchiduna/rustlings/tree/my_answer)

2. [google comprehesive](https://google.github.io/comprehensive-rust/)

十分推荐的 Rust 学习练习资源. 这门课程是谷歌开源出来的内部课程, 面对的是谷歌内部没有学习过 Rust 的高级工程师. 课程的形式是持续五天分上下午的讲解和对应的练习. 非常建议完成**每一个练习**. 这里面的每一个练习都相当有意思, 比较好的一点是, 这些练习都给出了相当精致(rusty)的参考解法. 你可以从中学习到 Rust 代码编写中的良好实践模式. 当然, 在学习的过程中, 翻阅[Rust std](https://doc.rust-lang.org/std/)也是必不可少的.
